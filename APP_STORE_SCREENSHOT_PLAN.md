# Safo App Store Screenshot Plan

Use this plan when capturing the first App Store screenshot set.

## Goal

Show Safo as:

- clear
- calm
- practical
- household-friendly
- smart, but not overwhelming

## Recommended Screenshot Order

### 1. Dashboard

Message:

`Know what needs attention today`

What to show:

- clean dashboard
- expiring food
- quick cooking recommendations
- a calm, not overloaded state

### 2. Pantry

Message:

`See what you already have at home`

What to show:

- organized pantry sections
- expiration info
- opened items
- warning indicators if useful but not too noisy

### 3. Shopping List

Message:

`Shop together without the chaos`

What to show:

- shared shopping list
- assigned item or bought state
- a clear, polished list view

### 4. Recipes

Message:

`Cook with what you already have`

What to show:

- recipe recommendations
- safe-for-you logic
- short cooking-time filter if it looks strong visually

### 5. Household

Message:

`Keep the whole household in sync`

What to show:

- household feed
- invite code area
- “Na teba čaká” / personal task section

## Optional Extra Screens

### 6. Kitchen Setup / Preferences

Message:

`Personalize Safo to your kitchen`

Use if the onboarding visual feels strong enough for store material.

### 7. Scan Flow

Message:

`Add food faster with scanning`

Use if the fridge scan or barcode flow looks polished enough for release marketing.

## Visual Rules

- prefer Slovak or English consistently within one screenshot set
- avoid placeholder or obviously demo-looking values when possible
- avoid very empty states unless the screenshot message is specifically about simplicity
- avoid very noisy dashboards
- keep household/member names clean and friendly
- prefer realistic pantry data over extreme edge cases

## Capture Notes

- capture on iPhone-sized layout
- make sure status bar, spacing, and bottom nav look stable
- avoid transitional states like spinners or snackbars
- if a card is too dense, simplify the data before capture

## Before Capturing

- load good tester sample data
- confirm no false allergy/intolerance warnings
- confirm localization on the chosen language set
- confirm app icon and splash are final enough
